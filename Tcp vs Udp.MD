Tcp:

1. 面向连接的传输（建立连接需三次握手，断开连接则需四次握手）
2. 端到端的通信
3. 高可靠性，确保传输数据的正确性，不出现丢失或乱序
4. 全双工方式传输
5. 采用字节流方式
6. 紧急数据传送功能



Tcp vs Udp

1. Tcp 协议面向连接，Udp 协议面向无连接
2. Tcp 协议传输速度慢，Udp 协议传输速度快
3. Tcp 协议保证数据顺序，Udp 协议不保证
4. Tcp 协议保证数据正确性，Udp 协议可能丢包
5. Tcp 协议对系统资源要求多，Udp 协议要求少

七层协议
1. 应用层（应用层，表示层，会话层）
2. 传输层（Tcp、Udp...）
3. 网络层
4. 数据链路层
5. 物理层


1. 一次要传送多少字节最适宜？
由于以太网的数据帧的长度必须在 46~1500 之间，这是由以太网的物理特性决定的。这个 1500 字节
被称为链路层的最大传输单元（MTU）（如果数据报大于 1500 字节，发送方的 IP 层就需要分片，接收方则要进行数据报的重组。），但并不包括链路层的首部和尾部的 18 个字节。1500 字节是网络层 IP 数据报的长度限制。

数据报的，又由于 Upd 数据包的首部占 8 字节，所以 Upd 数据报的数据区最大长度为 1472 字节。局域网
